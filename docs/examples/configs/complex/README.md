# Complex Config Examples

These configurations showcase advanced features and real-world scenarios for each extractor type. They include error handling, parallel processing, custom headers, and other production-ready settings.

## Files

- `api_complex.yaml` - Advanced REST API extraction with pagination, retries, and custom parsing
- `db_complex.yaml` - Advanced SQL database extraction with incremental loading and state management
- `file_complex.yaml` - Advanced file extraction with multiple formats and column selection
- `custom_complex.yaml` - Advanced custom extractor with complex logic

## Purpose

These configs demonstrate:
- Production-ready configurations
- Advanced extractor features
- Error handling and resilience patterns
- Performance optimizations
- Complex data transformation scenarios

## Usage

These configs are more complete examples that can be adapted for production use. They include best practices for reliability and performance.
